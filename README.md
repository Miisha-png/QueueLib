# Библиотека реализации очереди на кольцевом динамическом буфере

Легковесная и высокоэффективная библиотека на языке C для работы с **очередью на динамическом кольцевом буфере** (FIFO).

Библиотека оптимизирует потребление памяти, автоматически расширяя внутренний буфер при заполнении и сжимая его, если количество элементов становится значительно меньше текущей ёмкости.

---

## Основные возможности

* **Экономия памяти:** Автоматическое уменьшение размера буфера вдвое, если он заполнен менее чем на 25% (при превышении пороговой ёмкости).
* **Быстрые вычисления:** Добавление (`push`) и извлечение (`pop`) элементов выполняются за константное время $O(1)$ в амортизированном исчислении.
* **Безопасность:** Защита от утечек памяти, проверка на `NULL`-указатели и строгий контроль за границами кольцевого буфера.
* **Интеграция с Python:** Возможность написания готовой обертки на `ctypes` для вызова высокопроизводительных Си-методов управления очередью из Python-скриптов.

## Сборка и требования

### Требования к окружению

* Компилятор **GCC** (версии 9 или выше)
* Утилита **GNU Make**
* **Doxygen** (опционально, для генерации документации)

### Команды сборки

Все скомпилированные артефакты автоматически помещаются в директиву `build/`.

* **Собрать всё** (библиотеку и демо-приложение):

```bash
make

```

* **Собрать только разделяемую библиотеку (`.so`):**

```bash
make shared

```

* **Собрать и запустить демонстрационную программу:**

```bash
make run

```

## Справочник по API

### Основная структура данных

```c
typedef struct {
    int    *data;     // Указатель на динамический массив элементов
    size_t  head;     // Индекс первого элемента очереди
    size_t  tail;     // Индекс следующей свободной ячейки для вставки
    size_t  size;     // Текущее количество элементов в очереди
    size_t  capacity; // Текущая максимальная ёмкость буфера
} queue_t;

```

### Функции библиотеки

| Функция | Описание | Сложность |
| --- | --- | --- |
| `queue_create` | Создаёт и инициализирует новую очередь с заданной начальной ёмкостью. | $O(1)$ |
| `queue_destroy` | Уничтожает очередь и освобождает всю выделенную память. | $O(1)$ |
| `queue_push` | Добавляет элемент в конец очереди. При необходимости расширяет буфер вдвое. | $O(1)$ амортизированная |
| `queue_pop` | Извлекает и удаляет элемент из начала очереди. При необходимости сжимает буфер. | $O(1)$ амортизированная |
| `queue_peek` | Просматривает первый элемент очереди без его удаления. | $O(1)$ |
| `queue_is_empty` | Проверяет, пуста ли очередь. | $O(1)$ |
| `queue_is_full` | Проверяет, заполнена ли очередь (достигла ли лимита ёмкости). | $O(1)$ |
| `queue_get_size` | Возвращает текущее количество элементов в очереди. | $O(1)$ |
| `queue_resize` | *Внутренняя функция.* Перевыделяет память и выпрямляет кольцевой буфер. | $O(\text{size})$ |

## Управление памятью и логика ресайза

Библиотека берет на себя управление динамической памятью во избежание переполнения или избыточного выделения ресурсов:

* **Расширение (Grow):** Происходит внутри `queue_push`, когда `size == capacity`. Емкость удваивается (`RESIZE_GROW_FACTOR = 2`).
* **Сжатие (Shrink):** Происходит внутри `queue_pop`, когда текущий размер элементов `size` падает до **25%** и менее от общей емкости (`capacity / 4`), при условии, что текущая емкость больше минимального порога (`SHRINK_THRESHOLD = 4`). Емкость уменьшается в два раза.

Любое изменение размера приводит к "выпрямлению" кольцевого буфера — элементы копируются в новый линейный массив по порядку от `head` до `tail`, а сам индекс `head` сбрасывается в `0`.
